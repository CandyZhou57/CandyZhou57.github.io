项目中遇到的问题

项目中的settings.xml文件、maven中的settings.xml文件、本地的local repository是否需要保持一致。

项目中的.idea文件

项目中的.m2文件

使用idea启动项目，后台报：outofmemoryerror：Java heap space

通过在idea中设置相关参数

1.run-》edit configurations-》vm options

2.file-》settings-》heap

Error:java: OutOfMemoryError: insufficient memory

file->invalidate caches

Error running ‘App’: Command line is too long. Shorten command line for App or also for Spring Boot default configuration.

找到.idea文件下的workspace.xml文件，在component标签下添加

Error:java: Compilation failed: internal java compiler error

导致这个错误的原因主要是因为jdk版本问题，此处有两个原因，一个是编译版本不匹配，一个是当前项目jdk版本不支持

File ->Project Structure->Project Settings ->Project或使用快捷键Ctrl+Alt+shift+S打开项目的jdk配置：

查看此两处是否与目标jdk一致。

Error starting ApplicationContext. To display the conditions report re-run your application with ‘debug’ enabled.

Invocation of init method failed; nested exception is redis.clients.jedis.exceptions.JedisConnectionException: Could not get a resource from the pool

1.properties文件中的jedis配置有问题

2.未开启redis服务器

java.net.ConnectException: Connection refused: no further information

zookeeper连接不到导致

1.排查本地配置是否写的远程配置

2.核查本地zookeeper配置文件配置是否正确

3.核查是否为springboot集成redis–核查redis的配置文件

Caused by: redis.clients.jedis.exceptions.JedisConnectionException: java.net.ConnectException: Connection timed out: connect

原因：防火墙未开启指定端口导致

目标ip可能ping不通

redis连接超时

首先确保自己的server真正运行起来了

然后关闭防火墙，再重启服务

1.查看配置host和port

2.确保server正常运行

3.确保防火墙为关闭状态

4.代码中的socket有问题

1.依赖重复的问题

2.pom.xml中插件无法下载的问题
